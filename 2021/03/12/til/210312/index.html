Rubber Duck Team Activity Note

값의 할당(assignment)
일단 할당을 할때는 할당 연산자(=)를 사용한다.

Var score; //변수 선언
Score = 80; // 값의 할당

변수 선언과 값의 할당을 하나의 문(statement)으로 단축할 수 있다.
Var score =80; //변수의 선언 과 값의 할당

변수의 선언과 값의 할당의 실행시점이 다르다는 점을 명심해야한다.
변수 선언은 소스 코드가 순차적으로 실행되는 시점인 런타임 이전에 먼저 실행되지만 값의 할당은 순차적으로 실행되는 시점인 런타임에 실행된다.

값의 재할당?

Undefined 와 80은 어떤 변수도 값으로 가지고 있지 않다, 즉 어떤 식별자와도 연결되어 있지 않다. 이건 undefined와 80이 더 이상 필요하지 않는 것을 의미한다. 이러한 불필요한 값들은 garbage collector에 의해 메모리에서 해제된다. 하지만 언제 해제되는지는 모름.

원시타입

Number, string, boolean, undefined, null, symbol
숫자타입(number)은 한가지의 타입만 제공 실수(부동 수수점)만 제공. 정수는 없음
Infinity -infinity가 있는데 이건 8바이트를 넘어갈 경우 자동으로 할당하는 값
NaN는 숫자가 아님을 표시하는 값. 자바스크립트는 최대한 오류를 발생시키지 않으려고 한다.

문자열(string)은 다른 타입들과 다르게 값을 따옴표로 감싸는 이유는 식별자와 같은 토큰과 구별하기 위해서이다.
  일반 문자열에서는 줄바꿈이 허용되지 않는다. 자동으로 세미콜론을 추가하는 ASI 때문에 줄바꿈을 하면 문자열이 식별자로 변해버린다. 따라서 줄바꿈을 하려면 백슬래쉬를 써야한다.
하지만 탬플릿 리터럴(` `) 내에서는 이스케이프 시퀀스 백슬래쉬를 안써도 줄바꿈 가능
표현식 삽입은 ${}로 한다.


null은 의도적 부재. undefined는 자바스크립트 엔진이 변수를 초기화 할때 쓰는것이고, 변수에 값이 없다는 걸 할당하고 싶으면 null이라고 적는다

Naming conventintion
var person, $elem, _name, first_name, val1;

카멜 케이스, 스네이크 케이스, 파스칼 케이스, 헝가리안 케이스
변수나 함수에는 카멜 케이스/ 클래스나 생성자 함수에는 파스칼 케이스를 쓰는게 국룰

값(value)
식이 평가되어 생성된 결과를 말한다.
평가 = 식을 해석해서 값을 생성하거나 참조(reference)하는것
변수는 하나의 값을 할당하기 위해서 만든 메모리 공간 자체 또는 그것을 식별하기 위해 붙힌 이름

Literal
사람이 이해할 수 있는 문자 또는 기호를 사용해 값을 생성하는 표기방식

표현식(expression)
값으로 평가될 수 있는 statement이다. 즉 표현식이 평가되면 새로운 값을 생성하거나 기존 값을 참조한다.

문(statement)
프로그램을 구성 단위이자 최소 실행 단위

Token
문법적 의미를 가지며 문법적으로 더 이상 나눌 수 없는 코드의 기본 요소

표현식인 문과 표현식이 아닌 문
표현식인 문은 변수 할당이 가능하다/ 값처럼 사용할 수 있다.

표현식이 아닌 문은 변수 할당이 되지 않는다./ 값처럼 사용할 수 없다.

데이터 타입의 필요성
메모리에 값을 저장하기 위해서는 먼저 확보해야 하는 메모리 공간의 크기를 정해야 한다. 다시 말해서 몇 바이트의 메모리 공간을 사용해야 낭비와 손실없이 값을 저장할 수 있는지 알아야 한다.
즉 변수에 할당되는 값의 데이터 타입에 따라 확보해야 할 메모리 공간의 크기가 결정된다.(문자열과 숫자타입 이외에는 데이터 타입의 크기를 규정하고 있지 않다.)
그리고 메모리에서 읽어 들인 2진수를 어떻게 해석하느냐도 중요하다.
따라서
값을 저장할때 확보해야 하는 메모리 공간의 크기를 결정하기 위해서
값을 참조할때 한 번에 읽어 들어야 할 메모리 공간의 크기를 결정하기 위해
메모리에서 읽어들인 2진수를 어떻게 해석할지 결정하기 위해

Dynamic Typing
타입이 동적으로 바뀐다. 정적타이핑은 타입이 잘바뀌지 않는다
Int x = 1; //태어날때부터 타입을 가진다.
Var x = 1; //동적 타입
정적타입언어는 컴파일 시점에 타입 체크를 수행한다. 통과하지 못했으면 에러를 발생시키고 프로그램의 실행 자체를 막는다. 안정적이며 에러를 줄임
자바스크립트 변수는 선언이 아닌 할당에 의해 타입이 결정된다. 그리고 재할당에 의해 변수 타입은 언제든지 동적으로 변할 수 있다.

제어문(control flow statement)
주어진 조건에 따라 코드 블록을 실행하거나 반복 실행 할때 사용한다. 보통 코드는 위에서 아래로 진행하는데 제어문을 이를 방해할 수 있다.
그래서 가독성을 해친다. 가독성이 좋지않은 코드는 오류를 발생시킬 확률이 높다.
