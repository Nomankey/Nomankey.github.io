How to Call a Function

일반 함수로 호출 가능
New 연사자로 생성자 함수도 가능
객체 바인등 메서드화도 가능

ES6 사양에서 정의한 메서드는 인스턴스를 생성할 수 없는 non-constructor이다.

Arrow Function

화살표 함수는 일반 함수로만 호출이 가능하며 생성자 함수는 아니된다.
화살표 함수는 인스턴스를 생성할 수 없는 non-constructor
중복된 매개변수 이름을 선언할 수 없다
화살표 함수는 함수 자체의 this, arguments, super, new.target 바인딩을 가지지 않는다.

Rest Parameter

매개변수 이름앞에 …을 붙혀서 정의한 매개변수. Rest파라미터는 함수에 전달된 인수들의 목록을 배열로 전달받는다

Rest Parameters and arguments object

ES5에서는 매개변수의 개수를 정의할때 매개변수의 개수를 확정할 수 없는 가변 인자 함수의 경우 매개변수를 통해 인수를 전달받는게 불가능 하므로 arguments 객체를 이용해서 전달을 받았다.

Arguments 객체는 함수 호출 시 전달 된 인수(argument)들의 정보를 담고 있는 순회 가능한(iterable) 유사 배열 객체이며 함수 내부에서 지역변수처럼 활용 가능하다.

배열(array)

여러 값을 순차적으로 나열한 구조
array는 element와 element의 갯수를 나타내는 length 프로퍼티를 갖는다.

일반 객체와 배열을 구분하는 가장 명확한 차이는 값의 순서와 length 프로퍼티이다. 인덱스로 표현되는 값의 순서와 length 프로퍼티를 갖는 배열은 반복문을 통해 순차적으로 값에 접근하기 적합한 구조 이다.

배열의 장점은 순차적으로 element에 접근할 수 있다는 점이다.

왜 array를 만드는기?
Loop를 돌려서 같은 처리를 하기 위해서 만든다.
그렇기 위해서는 array내의 element들은 같은 타입을 가져야 수월하다.

자바스크립트에서의 배열은 일반적인 배열의 동작을 흉내 낸 특수한 객체이다. 배열 요소를 위한 각각의 메모리 공간은 동일한 크기를 갖지 않아도 되며, 연속적으로 이어져 있지 않을 수 도 있다. 배열의 요소가 연속적으로 이어져 있지 않는 배열은 희소 배열(sparse array)라 한다.

자바스크립트는 문법적으로 희소배열을 허용하지만 연속적인 값이라는 배열의 기본적인 개념과도 맞지 않고 성능에도 좋지않은 영향을 주기 때문에 하지 않는것이 좋다.
모던 자바스크립트는 타입이 일치하는 배열을 생성할때 일반적인 의미의 배열처럼 연속된 메모리 공간을 확보하는것으로 알려져 있다.
배열을 생성할때는 희소배열을 생성하지 않도록 하며 같은 타입의 요소를 연속적으로 배치하자

Array Method

Array 생성자 함수는 정적 메소드를 제공하며, 배열 객체의 프로토타입인 array.prototype은 프로토타입 메서드를 제공한다. 배열은 사용 빈도가 높은 자료 구조이므로 배열 메서드의 사용법을 잘 알아둘 필요가 있다.

array에는 원본 array를 직접 변경하는 method와 원본 array를 변경하지 않고 새로운 배열을 생성하여 반환하는 method가 있다.
