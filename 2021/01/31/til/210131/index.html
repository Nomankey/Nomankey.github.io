new fact i learned as pm in my first collaboration

위와 같이 제가 먼저 써놓은 코드를 팀원님들께서 fork를 해서 수정한후 팀장한테 보낼때
수정한 후의 코드가 구동이 안되면 팀장이 merge를 못하게 되어서 코드가 합쳐지지 않더라고요.

Ex)

위와 같은 경우에는 python 구동이 불가능한 코드이기 때문에 merge가 안되었습니다.

다행히 github내에서 confirm merge 버튼옆에 solve confliction 버튼이 있어서 GitHub 홈페이지 내에서 코드를 수정할 수 있습니다.

Ex)

위와 같이 바꾸면 웹상에서 바꾸면
confliction이 사라지기 때문에 팀장이 성공적으로 merge가 가능해집니다.

결론!
이런일이 생기지 않도록하기위해서 본격적인 협업을 할때는 팀원당 1하나의 파일을 담당할것 같습니다.
한가지 파일을 여러명이서 수정하면 confliction문제가 불가피하기때문에

그리고 혹시나 여러명이 한파일을 담당할때는 코드를 합쳤을때 구동이 가능한 쪽으로 보내는게 좋을것 같습니다.
