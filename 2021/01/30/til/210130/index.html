TIL 210130

learned a process how to collabor as a team
The description below is the process of how PM and team member work together on git

팀장 -1

  • Either GitHub
  • Make new repository
    • Initialize this repository with
      • Add a README.file
      • Add .gitignore
  • 터미널로 이동한다.
  • $git clone <리포주소>
  • $cd clone -anything
    • 클론 파일 이름임..
  • $git flow init
  • $git branch
    • branch가 main에서 develop으로 이동 확인
  • $touch index.html
  • $ls
    • 만든 파일 확인
  • $git status
    • Index.html 가 untracked file 이라고 뜬다.
  • $git commit
    • feat: create index.html
    • TODO
    • -semantic elements
    • -web standards
    • -separate css, js files in to static
  • $git push -u origin develop
    • branch가 main이면 -u가 필요없지만 branch 가 develop 이기에 필요하다.

      팀원 - 1

  • 팀장 repository 방문 & Fork
  • $git clone <팀장 리포주소>
  • $cd clone -anything
  • $ls
  • $git branch
    • branch에 main만 있는걸 확인한다.
  • $git flow init
    • branch가 develop에 있는 것 확인
  • $ls
    • index.html 확인후 팀장 지령 숙지
  • 팀장 GitHub 방문하고나서
    • 팀장 repository
    • Issues
    • New issue
    • 팀원으로서 팀장 repository 글을 남긴다 ex) 내가 한당, 내가 찜한당

      팀장 -2

    • 답을 한다
    • Label을 준다
      #팀원 - 2
  • 팀장한테 label을 받으면 confirm, 바로 작업 ㄲ
  • 터미널로 돌아온다
  • $ls
  • $git flow feature start <작업이름>
    • Ex) split - static
  • $vi index.html
    • 작업 ㄲ
  • $git status -uall
  • $git add index.html
  • $git status
    • 파일 올라온거 확인
  • $git commit
    • 팀장 repositroy에서 issue 남길때 받은 숫자를 남긴다
      • Ex) feat: separate css, js in to static/
      • Sovled: #1
  • $git flow feature finish <작업이름>
    • Ex) split-static
  • $git push origin develop
    • 팀원은 develop으로 push한다. 안그러면 큰일남…
  • Clone-anything repository로 복귀
  • fork로 따서 작업한게 반영되었는지 확인
  • Pull request를 누른다.
    • branch가 develop인지 확인
  • Open a pull request화면에서 팀원의 develop에서 팀장의 develop으로 갔는지 확인.
  • Create pull request

    팀장 - 3

  • 팀원이 받은 pull request 확인
    • 후에 피드백을 남긴다. Ex) title text를 추가하게
    • 이때는 pull request가 open 되어있다

      팀원 - 3

  • 팀원은 피드백 수용후 다시 작업
  • $git add index.html
    • 작업한걸 다시 올린다
  • $git commit
    • feat: fill the title text
  • $git push origin develop
    • 팀장의 pull request가 open일때 develop에서 push하면 무조건 팀장의 pull request로 push된다.
    • 프로젝트의 효율성을 위해 팀장은 빨리 확인하고 pull request를 닫는게 중요
    • 안그러면 다음 작업으로 못넘어감 ㅠㅜ

      팀장 - 4

  • 확인후에 confirm merge
  • 팀장의 develop에 팀원의 내용이 들어간다

    팀원 - 4

  • $git remote -v
  • $git remote add pmorigin <팀장 리포 주소>
  • $git remote -v
  • $git pull pmorigin develop
    • 팀원이 작업한것들을 확인 가능하다.
