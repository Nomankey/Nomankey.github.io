Browser Rendering Process

구글 V8 JavaScript Engine으로 빌드된 JavaScript Runtime Environment인 Node.Js의 등장으로 웹브라우저에서 벗어나서 servier side development가 가능해진 multi-purpose language가 되었지만 여전히 가장 많이 사용되는 분야는 웹 브라우저 환경에서 동작하는 웹페이지/애플리케이션 분야이다.

  • 대부분의 프로그래밍 언어는 브라우저가 아니라 Operating System이나 Virtual Machine위에서 실행된다.

자바스크립트가 브라우저에서 작동하느다는 것을 알았으니 어떻게 parse하여 browser에 redering하는지 알아본다.

  • parse는 해석을 의미한다.

  • 프로그래밍 언어에 맞게 작성된 텍스트 문서를 읽어 들여 실행하기 위해서 문자열을 토큰으로 분해하고 거기에 문법적 의미와 구조를 반영하여 트리 구조의 자료구조인 parse tree/syntex tree 를 생성하는 과정을 말함

  • Rendering

  • Html, css, javascript를 parsing 하여 브라우저에 시각적으로 출력하는 것.

Request and Response

브라우저의 핵심 기능은 필요한 리소스를 server에 request하고 server로부터 response를 받아서 rendering 하는 것이다.
서버에 request를 전송하기 위해 브라우저는 주소창을 제공한다. 브라우저 주소창에 URL을 입력하고 엔터 키를 누르면 URL의 호스트 이름이 DNS를 통해 IP주소로 변환되고 이 IP 주소를 갖는 서버에게 요청을 전송한다.

HTTP 1.1과 HTTP 2.0

HTTP(HyperText Transfer Protocol )는 웹에서 브라우저와 서버가 통신을 하기 위한 protocol(규약)이다.

HTTP/1.1은 connection 당 하나의 응답을 한다. 즉 여러개의 요청을 한번에 전송할 수 없다.
Resource request가 개별적으로 전송되고 response또한 개별적으로 전송된다. 요청할 resource에 비례하여 응답시간이 길어진다.

반면에 HTTP/2는 connection 당 여러 개의 요청과 응답, 즉 다중 요청/응답이 가능하다. HTTP 1.1에 비해서 속도가 약 50%정도 빠르다.

HTML parsing and DOM production

브라우저의 요청에 의해 서버가 응답한 HTML 문서는 문자열로 이루어진 순수한 text이다. Browser Rendering Engine은 응답받은 HTML 문서를 파싱하여 브라우저가 이해할 수 있는 DATA STRUCTURE인 DOM(Document Object Model)을 생성한다.
DOM은 문서를 parsing 한 결과물이다.

CSS 파싱과 CSSOM 생성

Redering Engine은 HTML을 처음부터 한 줄 씩 순차적으로 파싱하여 DOM을 생성해 나간다. 그러나 CSS Link를 로드하는 link 태그나 style 태그를 만나면 DOM 생성을 일시중지한다.

그러고 나서 link 태그의 href attribute(속성)에 지정된 CSS 파일을 서버에 요청하여(바이트 -> 문자 -> 토큰 -> 노드 -> CSSOM)과 같이 HTML과 똑같은 과정을 거쳐 해석하여 CSSOM(CSS Object Model)을 생성한다.그리고 CSS 파싱이 완료되면 다시 HTML파싱을 시작한다.

Render Tree 생성

DOM과 CSSOM을 생성했으면 이들은 랜더링을 위해 Render Tree로 결합된다.
Render Tree는 Redering을 하기위한 tree shaped data structure이다. 따라서 브라우저 화면에 rendering 하지 않는 node는 포함하지 않는다.

이후 완성된 render tree는 각 HTML 요소의 레이아웃을 계산하는 데 사용되며 브라우저 화면에 픽셀을 렌더링하는 페인팅 처리에 입력된다

렌더링 작업은 반복되서 실행될 수 있다.

  • 자바스크립트에 의한 노드 추가 또는 삭제
  • 브라우저 창의 리사이징
  • HTML요소의 크기를 조정

위와 같은 작업은 리렌더링에 비용이 많이 드는 작업이므로 가급적 삼가하도록 한다.

JavaScript Parsing and Execution

HTML parsing의 결과물로 생성된 DOM은 HTML 문서의 구조와 정보 뿐만 아니라 요소와 스타일도 변경할 수 있는 DOM API를 제공한다. DOM API를 사용하면 이미 생성된 DOM을 동적으로 조작할 수 있다.

Rendering engine으로 부터 제어권을 넘겨받은 javascript engine은 javascript code를 파싱하기 시작하는데, 자바스크립트 엔진은 자바스크립트를 해석하여 AST(Abstract Syntax Tree, 추상 문법 트리)를 생성한다.그러고나서 AST를 기반으로 인터프리터가 실행할 수 있는 중간 코드(intermediate code)인 바이트코드를 생성하여 실행한다.

파싱으로 결과로 생긴 AST는 인터프리터가 실행할 수 있는 중간 코드인 바이트코드로 변환되고 인터프리터에 의해 실행된다.

Reflow and Repaint

자바스크립트 코드에 DOM이나 CSSOM을 변경하는 DOM API가 사용된 경우 DOM이나 CSSOM이 변경된다. DOM과 CSSOM은 다시 Render Tree로 결합되고 Render Tree를 기반으로 레이아웃과 페인트 과정을 통해 브라우저 화면에 다시 렌더링 한다

리플로우는 레이아웃 계산을 다시하는것을 의미하므로 노드 추가/삭제, 요소의 크기/위치 변경, 윈도우 리사이징 등 레이아웃에 영향을 주는 변경이 발생한 경우에 한하여 실행된다.

script tag의 async/ defer attribute

자바스크립트 파싱에 의한 DOM 생성이 중단되는 문제를 근본적으로 해결하기 위한 attribute들이다.

async와 defer attribute는 다음과 같이 src 어트리뷰트를 통해 외부 자바스크립트 파일을 로드하는 경우에만 사용할 수 있다.

async attribute
HTML parsing 과 외부 자바스크립트 파일의 로드가 비동기적으로 동시에 진행된다. 단, 자바스크립트의 파싱과 실행은 자바스크립트 파일의 로드가 완료된 직후 진행되며 이때 HTML parsing이 중단된다.

defer attribute
async와 마찬가지로 비동기적으로 동시에 진행된다. 하지만 DOM이 생성된 직후에 진행된다. 따라서 DOM이 완료된 이후에 실행되어야 할 자바스크립트에 유용하다.

What is Array?

배열(array)는 여러개의 값을 순차적으로 나열한 자료 구조(Data Structure).

ex) const arr = [‘apple’, ‘banana’, ‘orange’]

배열이 가지고 있는 값은 요소(element)이다. 자바스크립트에서 모든 값은 배열이 될 수 있다.

ex)
arr[0] // apple
arr[1] // banana
arr[2] // orange

위와 같이 element에 접근할때는 대괄호 표기법을 갖는다.

ex)
arr.length // 3

배열은 또한 요소의 개수, 즉 배열의 길이를 나타내는 length 프로퍼티를 갖는다.

ex)
for (let i = 0; I < arr.length; i++){
console.log(arr[i]);
}

배열은 인덱스와 length 프로퍼티를 갖기 때문에 for문을 통해 순차적으로 요소에 접근할 수 있다.

또한 자바스크립트는 배열(array)라는 타입이 없기 때문에 배열은 객체 타입이다.

ex)
const arr = [1, 2, 3];

arr.constructor === Array // true
Object.getPrototypeOf(arr) === Array.prototype // true

배열은 배열 리터럴, array 생성자 함수, array.of, array.from 메서드로 생성가능
배열의 프로토타입 객체는 array.prototype

Array in JavaScript is not an Array!

자료 구조(data structure)에서 배열이란 동일한 크기의 메모리 공간이 빈틈이 없이 연속적으로 나열된 구조
=> 밀집 배열(dense array)

인덱스를 통해 단 한번의 연산으로 임의의 요소에 접근(random access) 하여 효율적으로 고속으로 동작한다.

검색 대상의 메모리 주소 = 배열의 시작 메모리 주소 + 인덱스*요소의 바이트 수
• 인덱스가 0인 요소의 메모리 주소: 1000 + 0 * 8 = 1000
• 인덱스가 1인 요소에 메모리 주소: 1000 + 1 * 8 = 1008
• 인덱스가 2인 요소에 메모리 주소: 1000 + 2 * 8 = 1016

인덱스를 통해 효율적으로 요소에 접근할 수 있지만 정렬되지 않은 배열에서 특정한 요소를 검색할때는 array의 모든 요소를 처음부터 특정 요소를 발견할 때까지 차례대로 검색(linear search)를 해야한다.
array에 element를 삽입하거나 삭제할때 element를 연속적으로 유지하기 위해서는 element를 이동시켜야 한다

자바스크립트의 배열은 지금까지와의 배열과는 다르게 array의 element를 위한 각각 메모리 공간의 크기가 동일하지 않아도 되고 연속적으로 이어져 있지도 않다.
=> 배열의 요소가 연속적으로 이어져 있지 않은 배열을 희소 배열(sparse array)이라 한다.

위처럼 자바스크립 배열은 인덱스를 나타내는 문자열을 프로퍼티 키로 가지며, length 값을 프로퍼티로 갖는 특수한 객체이다. 자바스크립트 array elements 는 사실상 property value 이다.

일반적인 배열(array list)과 자바스크립트 배열(linked list)의 장단점

  • 일반적인 배열은 인덱스로 요소에 빠르게 접근이 가능하며 특정 요소를 검색하거나 요소를 삽입 삭제할때는 효율적이지 않다

  • 자바스크립트 배열은 해시 테이블로 구현된 객체이기에 인덱스로 요소에 접근하는 경우에는 일반적인 배열보다 성능이 느릴 수 밖에 없지만, 특정 요소를 검색하거나 요소를 삽입 또는 삭제하는 경우에는 일반적인 배열보다는 빠른 성능을 기대할 수 있다.

Binary Tree

기본적으로 Binary Tree는 edges로 연결되어있는 node들로 구성되어있다.

Binary Tree t가 있다고 가정을 해보자.
이 tree는 비어있거나 2개의 자식요소 (left subtree, right subtree)를 가진 단일 root 요소를 가질 것이다.
Binary라는 이름에서 알 수 있듯이 , binary tree에서는 최대 2개의 자식을 가질 수 있다.

  • Binday Tree Diagram에서 root는 맨 위에 있고 subtrees는 아래로 갈라져 나온다
  • Node의 left subtree 와 right subtree는 자식이며 node는 sub tree들의 부모이다.
  • root도 아니며 자식도 없는 node는 leaf라고 불린다.
  • Binary tree의 root node는 모든 node들의 조상이다.
  • 어떠한 node a 가 어떠한 node b의 조상이라면, a로부터 b까지의 경로는 a에서 b까지의 혈육 사슬을 따른 길이다.
  • tree의 높이는 root와 가장 멀리있는 leaf 사이의 edge 수이다. 더 기술적으로 표기를 하자면
    • 1+max(height(leftSubtree), height(rightSubtree))

How to Call a Function

일반 함수로 호출 가능
New 연사자로 생성자 함수도 가능
객체 바인등 메서드화도 가능

ES6 사양에서 정의한 메서드는 인스턴스를 생성할 수 없는 non-constructor이다.

Arrow Function

화살표 함수는 일반 함수로만 호출이 가능하며 생성자 함수는 아니된다.
화살표 함수는 인스턴스를 생성할 수 없는 non-constructor
중복된 매개변수 이름을 선언할 수 없다
화살표 함수는 함수 자체의 this, arguments, super, new.target 바인딩을 가지지 않는다.

Rest Parameter

매개변수 이름앞에 …을 붙혀서 정의한 매개변수. Rest파라미터는 함수에 전달된 인수들의 목록을 배열로 전달받는다

Rest Parameters and arguments object

ES5에서는 매개변수의 개수를 정의할때 매개변수의 개수를 확정할 수 없는 가변 인자 함수의 경우 매개변수를 통해 인수를 전달받는게 불가능 하므로 arguments 객체를 이용해서 전달을 받았다.

Arguments 객체는 함수 호출 시 전달 된 인수(argument)들의 정보를 담고 있는 순회 가능한(iterable) 유사 배열 객체이며 함수 내부에서 지역변수처럼 활용 가능하다.

배열(array)

여러 값을 순차적으로 나열한 구조
array는 element와 element의 갯수를 나타내는 length 프로퍼티를 갖는다.

일반 객체와 배열을 구분하는 가장 명확한 차이는 값의 순서와 length 프로퍼티이다. 인덱스로 표현되는 값의 순서와 length 프로퍼티를 갖는 배열은 반복문을 통해 순차적으로 값에 접근하기 적합한 구조 이다.

배열의 장점은 순차적으로 element에 접근할 수 있다는 점이다.

왜 array를 만드는기?
Loop를 돌려서 같은 처리를 하기 위해서 만든다.
그렇기 위해서는 array내의 element들은 같은 타입을 가져야 수월하다.

자바스크립트에서의 배열은 일반적인 배열의 동작을 흉내 낸 특수한 객체이다. 배열 요소를 위한 각각의 메모리 공간은 동일한 크기를 갖지 않아도 되며, 연속적으로 이어져 있지 않을 수 도 있다. 배열의 요소가 연속적으로 이어져 있지 않는 배열은 희소 배열(sparse array)라 한다.

자바스크립트는 문법적으로 희소배열을 허용하지만 연속적인 값이라는 배열의 기본적인 개념과도 맞지 않고 성능에도 좋지않은 영향을 주기 때문에 하지 않는것이 좋다.
모던 자바스크립트는 타입이 일치하는 배열을 생성할때 일반적인 의미의 배열처럼 연속된 메모리 공간을 확보하는것으로 알려져 있다.
배열을 생성할때는 희소배열을 생성하지 않도록 하며 같은 타입의 요소를 연속적으로 배치하자

Array Method

Array 생성자 함수는 정적 메소드를 제공하며, 배열 객체의 프로토타입인 array.prototype은 프로토타입 메서드를 제공한다. 배열은 사용 빈도가 높은 자료 구조이므로 배열 메서드의 사용법을 잘 알아둘 필요가 있다.

array에는 원본 array를 직접 변경하는 method와 원본 array를 변경하지 않고 새로운 배열을 생성하여 반환하는 method가 있다.

유사 배열 객체 (Array Like Objects)

var yoosa = {
    0: 'a',
    1: 'b',
    2: 'c',
    //length: 3
};

키가 숫자고 length라는 속성을 가지고 있다. 배열도 객체라는 특성을 이용한 trick d이다.

따라서 array 처럼 yoosa[0], yoosa[1], yoosa[2] 같은 것을 모두 활용 가능하다.
