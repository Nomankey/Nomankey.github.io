Binary Tree

기본적으로 Binary Tree는 edges로 연결되어있는 node들로 구성되어있다.

Binary Tree t가 있다고 가정을 해보자.
이 tree는 비어있거나 2개의 자식요소 (left subtree, right subtree)를 가진 단일 root 요소를 가질 것이다.
Binary라는 이름에서 알 수 있듯이 , binary tree에서는 최대 2개의 자식을 가질 수 있다.

  • Binday Tree Diagram에서 root는 맨 위에 있고 subtrees는 아래로 갈라져 나온다
  • Node의 left subtree 와 right subtree는 자식이며 node는 sub tree들의 부모이다.
  • root도 아니며 자식도 없는 node는 leaf라고 불린다.
  • Binary tree의 root node는 모든 node들의 조상이다.
  • 어떠한 node a 가 어떠한 node b의 조상이라면, a로부터 b까지의 경로는 a에서 b까지의 혈육 사슬을 따른 길이다.
  • tree의 높이는 root와 가장 멀리있는 leaf 사이의 edge 수이다. 더 기술적으로 표기를 하자면
    • 1+max(height(leftSubtree), height(rightSubtree))

How to Call a Function

일반 함수로 호출 가능
New 연사자로 생성자 함수도 가능
객체 바인등 메서드화도 가능

ES6 사양에서 정의한 메서드는 인스턴스를 생성할 수 없는 non-constructor이다.

Arrow Function

화살표 함수는 일반 함수로만 호출이 가능하며 생성자 함수는 아니된다.
화살표 함수는 인스턴스를 생성할 수 없는 non-constructor
중복된 매개변수 이름을 선언할 수 없다
화살표 함수는 함수 자체의 this, arguments, super, new.target 바인딩을 가지지 않는다.

Rest Parameter

매개변수 이름앞에 …을 붙혀서 정의한 매개변수. Rest파라미터는 함수에 전달된 인수들의 목록을 배열로 전달받는다

Rest Parameters and arguments object

ES5에서는 매개변수의 개수를 정의할때 매개변수의 개수를 확정할 수 없는 가변 인자 함수의 경우 매개변수를 통해 인수를 전달받는게 불가능 하므로 arguments 객체를 이용해서 전달을 받았다.

Arguments 객체는 함수 호출 시 전달 된 인수(argument)들의 정보를 담고 있는 순회 가능한(iterable) 유사 배열 객체이며 함수 내부에서 지역변수처럼 활용 가능하다.

배열(array)

여러 값을 순차적으로 나열한 구조
array는 element와 element의 갯수를 나타내는 length 프로퍼티를 갖는다.

일반 객체와 배열을 구분하는 가장 명확한 차이는 값의 순서와 length 프로퍼티이다. 인덱스로 표현되는 값의 순서와 length 프로퍼티를 갖는 배열은 반복문을 통해 순차적으로 값에 접근하기 적합한 구조 이다.

배열의 장점은 순차적으로 element에 접근할 수 있다는 점이다.

왜 array를 만드는기?
Loop를 돌려서 같은 처리를 하기 위해서 만든다.
그렇기 위해서는 array내의 element들은 같은 타입을 가져야 수월하다.

자바스크립트에서의 배열은 일반적인 배열의 동작을 흉내 낸 특수한 객체이다. 배열 요소를 위한 각각의 메모리 공간은 동일한 크기를 갖지 않아도 되며, 연속적으로 이어져 있지 않을 수 도 있다. 배열의 요소가 연속적으로 이어져 있지 않는 배열은 희소 배열(sparse array)라 한다.

자바스크립트는 문법적으로 희소배열을 허용하지만 연속적인 값이라는 배열의 기본적인 개념과도 맞지 않고 성능에도 좋지않은 영향을 주기 때문에 하지 않는것이 좋다.
모던 자바스크립트는 타입이 일치하는 배열을 생성할때 일반적인 의미의 배열처럼 연속된 메모리 공간을 확보하는것으로 알려져 있다.
배열을 생성할때는 희소배열을 생성하지 않도록 하며 같은 타입의 요소를 연속적으로 배치하자

Array Method

Array 생성자 함수는 정적 메소드를 제공하며, 배열 객체의 프로토타입인 array.prototype은 프로토타입 메서드를 제공한다. 배열은 사용 빈도가 높은 자료 구조이므로 배열 메서드의 사용법을 잘 알아둘 필요가 있다.

array에는 원본 array를 직접 변경하는 method와 원본 array를 변경하지 않고 새로운 배열을 생성하여 반환하는 method가 있다.

Rubber Duck Team Activity Note

값의 할당(assignment)
일단 할당을 할때는 할당 연산자(=)를 사용한다.

Var score; //변수 선언
Score = 80; // 값의 할당

변수 선언과 값의 할당을 하나의 문(statement)으로 단축할 수 있다.
Var score =80; //변수의 선언 과 값의 할당

변수의 선언과 값의 할당의 실행시점이 다르다는 점을 명심해야한다.
변수 선언은 소스 코드가 순차적으로 실행되는 시점인 런타임 이전에 먼저 실행되지만 값의 할당은 순차적으로 실행되는 시점인 런타임에 실행된다.

값의 재할당?

Undefined 와 80은 어떤 변수도 값으로 가지고 있지 않다, 즉 어떤 식별자와도 연결되어 있지 않다. 이건 undefined와 80이 더 이상 필요하지 않는 것을 의미한다. 이러한 불필요한 값들은 garbage collector에 의해 메모리에서 해제된다. 하지만 언제 해제되는지는 모름.

원시타입

Number, string, boolean, undefined, null, symbol
숫자타입(number)은 한가지의 타입만 제공 실수(부동 수수점)만 제공. 정수는 없음
Infinity -infinity가 있는데 이건 8바이트를 넘어갈 경우 자동으로 할당하는 값
NaN는 숫자가 아님을 표시하는 값. 자바스크립트는 최대한 오류를 발생시키지 않으려고 한다.

문자열(string)은 다른 타입들과 다르게 값을 따옴표로 감싸는 이유는 식별자와 같은 토큰과 구별하기 위해서이다.
  일반 문자열에서는 줄바꿈이 허용되지 않는다. 자동으로 세미콜론을 추가하는 ASI 때문에 줄바꿈을 하면 문자열이 식별자로 변해버린다. 따라서 줄바꿈을 하려면 백슬래쉬를 써야한다.
하지만 탬플릿 리터럴(` `) 내에서는 이스케이프 시퀀스 백슬래쉬를 안써도 줄바꿈 가능
표현식 삽입은 ${}로 한다.


null은 의도적 부재. undefined는 자바스크립트 엔진이 변수를 초기화 할때 쓰는것이고, 변수에 값이 없다는 걸 할당하고 싶으면 null이라고 적는다

값(value)
식이 평가되어 생성된 결과를 말한다.
평가 = 식을 해석해서 값을 생성하거나 참조(reference)하는것
변수는 하나의 값을 할당하기 위해서 만든 메모리 공간 자체 또는 그것을 식별하기 위해 붙힌 이름

Literal
사람이 이해할 수 있는 문자 또는 기호를 사용해 값을 생성하는 표기방식

표현식(expression)
값으로 평가될 수 있는 statement이다. 즉 표현식이 평가되면 새로운 값을 생성하거나 기존 값을 참조한다.

문(statement)
프로그램을 구성 단위이자 최소 실행 단위

Token
문법적 의미를 가지며 문법적으로 더 이상 나눌 수 없는 코드의 기본 요소

표현식인 문과 표현식이 아닌 문
표현식인 문은 변수 할당이 가능하다/ 값처럼 사용할 수 있다.

표현식이 아닌 문은 변수 할당이 되지 않는다./ 값처럼 사용할 수 없다.

데이터 타입의 필요성
메모리에 값을 저장하기 위해서는 먼저 확보해야 하는 메모리 공간의 크기를 정해야 한다. 다시 말해서 몇 바이트의 메모리 공간을 사용해야 낭비와 손실없이 값을 저장할 수 있는지 알아야 한다.
즉 변수에 할당되는 값의 데이터 타입에 따라 확보해야 할 메모리 공간의 크기가 결정된다.(문자열과 숫자타입 이외에는 데이터 타입의 크기를 규정하고 있지 않다.)
그리고 메모리에서 읽어 들인 2진수를 어떻게 해석하느냐도 중요하다.
따라서
값을 저장할때 확보해야 하는 메모리 공간의 크기를 결정하기 위해서
값을 참조할때 한 번에 읽어 들어야 할 메모리 공간의 크기를 결정하기 위해
메모리에서 읽어들인 2진수를 어떻게 해석할지 결정하기 위해

Dynamic Typing
타입이 동적으로 바뀐다. 정적타이핑은 타입이 잘바뀌지 않는다
Int x = 1; //태어날때부터 타입을 가진다.
Var x = 1; //동적 타입
정적타입언어는 컴파일 시점에 타입 체크를 수행한다. 통과하지 못했으면 에러를 발생시키고 프로그램의 실행 자체를 막는다. 안정적이며 에러를 줄임
자바스크립트 변수는 선언이 아닌 할당에 의해 타입이 결정된다. 그리고 재할당에 의해 변수 타입은 언제든지 동적으로 변할 수 있다.

제어문(control flow statement)
주어진 조건에 따라 코드 블록을 실행하거나 반복 실행 할때 사용한다. 보통 코드는 위에서 아래로 진행하는데 제어문을 이를 방해할 수 있다.
그래서 가독성을 해친다. 가독성이 좋지않은 코드는 오류를 발생시킬 확률이 높다.

Runtime Error

When it happens?

Runtime Error는 프로그래밍이 성공적으로 compile된 이후에 프로그램이 실행되는 도중에 발생한다.

개발을 시작하는 사람들이 가장 많이 마주하게 될 문제들 중 하나이다. 이것이 발생하는 주된이유들은

  • segmentation fault: 메모리에 허용되지 않는 접속을 할때
  • 10^8 보다 많은 array를 선언했을때
  • 0으로 modulus를 나누거나 가져올때
  • GDB를 사용하면 runtime error를 보다 쉽게 디버깅이 가능하다.

runtime error는 많은 경우에 bug라고 불리며 소프웨어가 출시되기전 debugging 과정에서 많이 발생한다.
public에 프로그램이 출시된 이후에 runtime error가 발생하면 개발자들은 error를 고치기 위해 패치나 작은 업데이트등은 진행한다.
