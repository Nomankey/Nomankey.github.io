The description below is the process of how PM and team member work together on git

팀장 -1

  • Either GitHub
  • Make new repository
    • Initialize this repository with
      • Add a README.file
      • Add .gitignore
  • 터미널로 이동한다.
  • $git clone <리포주소>
  • $cd clone -anything
    • 클론 파일 이름임..
  • $git flow init
  • $git branch
    • branch가 main에서 develop으로 이동 확인
  • $touch index.html
  • $ls
    • 만든 파일 확인
  • $git status
    • Index.html 가 untracked file 이라고 뜬다.

  • $git push -u origin develop

  • 팀장 repository 방문 & Fork

  • 팀원이 받은 pull request 확인
    • 후에 피드백을 남긴다. Ex) title text를 추가하게
    • 이때는 pull request가 open 되어있다

      팀원 - 3

  • 팀원은 피드백 수용후 다시 작업
  • $git add index.html
    • 작업한걸 다시 올린다
  • $git commit
    • feat: fill the title text
  • $git push origin develop
    • 팀장의 pull request가 open일때 develop에서 push하면 무조건 팀장의 pull request로 push된다.
    • 프로젝트의 효율성을 위해 팀장은 빨리 확인하고 pull request를 닫는게 중요
    • 안그러면 다음 작업으로 못넘어감 ㅠㅜ

  • 확인후에 confirm merge

  • $git remote -v
  • $git pull pmorigin develop
    • 팀원이 작업한것들을 확인 가능하다.

List, Tuple

List
animals = [‘’, ‘’, ‘’]
Tuple
animals= (‘’, ‘’, ‘’)

List

  • List 는 선언과 동시에 값을 채워 넣을 수 있다.
  • 요소를 추가할 수 있다 (.append())
  • 특정한 위치에 원하는 값을 추가할 수 있다. lang.insert(1, “c”) print(lang)
  • 특정 요소를 삭제할 수도 있다. lang.remove(“golang”) print(lang)
  • 리스트에 있던 값을 빼낼 수도 있다. java = lang.pop(2) print(lang) print(java)
  • 리스트를 정렬. numbers = [2, 1, 4, 3] print(numbers) /t numbers.sort() print(numbers)
  • 리스트를 역순으로 출력. numbers = [2, 1, 4, 3] numbers.reverse() print(numbers)
  • 특정 값의 위치 출력. index_of_two = numbers.index(2) print(index_of_two)
  • 리스트끼리 더할 때는 extend. numbers += [5, 6] print(numbers) numbers.extend([7, 8]) print(numbers)

Membership Operator

in - ‘cat’ in [‘fox’, ‘dog’, ‘cat’] => True
not in - ‘wolf’ not in [‘fox’, ‘dog’, ‘cat’] => True

Tuple

  • 괄호를 사용해서 선언한다. tuple1 = (1, 2, 3, 4)
  • 삭제나 추가가 불가능.
  • tuple끼리 더하거나 반복은 가능. tuple2 = (5, 6) print(tuple1 + tuple2) /t print(tuple1 * 3)
  • 값을 편하게 바꿀 수 있다.
  • 함수에서 하나 이상의 값을 반환할때 사용한다

    Dictionary

  • 매핑타입의 묶음 자료형이다.
  • Key와 value로 구성되어 있다.
  • Hash table 을 이용하여 구현 -> 일정한 속도 -> 키의 수가 맞지 않아도 일정한 탐색속도 유지

dictionary 선언

dict1 = {} print(dict1)
dict2 = {‘ja’:0, ‘chuk’:1, ‘in’:2}
dict3 = dict(ja=0,chuk=1,in=2)
dict4 = dict([(‘ja’,0), (‘chuk’,1),(‘in’,2)])
